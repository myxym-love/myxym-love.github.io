将hexo静态博客资源部署到阿里云服务器

1.WHY？

大多数人使用hexo搭建个人博客，基本 都是部署在Github Page或者Gitee Page上的,相较于Github Page,部署在Gitee Page上的人应该会更多点。因为Github Page毕竟是国外的Github网站提供给每一个加入GitHub的一个静态服务器,国内用户访问起来的体验感实在是太差了。而Gitee Page是国内Gitee提供的，国内用户访问起来想对于来说体验感好一点。但还有没有其他方式部署我们的个人博客呢？当然有！只要你有一台属于自己的云服务器，那么也可以将个人博客部署在云服务器上。云服务器的购买途径有很多，在这里，我推荐使用阿里云服务器或者腾讯云服务器。

2.HOW？

1.本地的准备

为了能够是使本地的项目与远程的git联系起来， 需要在本地电脑上生成密钥，使用如下命令在cmd终端中输入

ssh-keygen

一路按回车就行，生成的公钥和私钥默认的路径是你用户下.ssh文件夹下,例如我的路径C:\Users\ASUS\.ssh，id_rsa是私钥,id_rsa.pub是公钥，我们需要用到公钥里的内容，下面的内容会提到。

2.云服务器的准备

在云服务器上我们需要安装git:

yum install git

安装好后，我们新建一个git用户：

adduser git

添加git用户权限：

chmod 740 /etc/sudoers

然后用vim编辑器打开文件编辑

vim /etc/sudoers

找到如下位置，并添加

root   ALL=(ALL)     ALL #文件内的位置
git    ALL=(ALL)     ALL #添加的内容

把权限更改回去

chmod 400 /etc/sudoers

设置 git 账户密码：

sudo passwd git

Linux输入密码是不会显示，输入好了按回车就行。

切换至git用户，创建 ~/.ssh 文件夹和 ~/.ssh/authorized_keys 文件，并赋予相应的权限

su git
mkdir  ~/.ssh
vim  ~/.ssh/authorized_keys

将我们在win10中生成的id_rsa.pub文件中的公钥复制到authorized_keys中。

接着，输入一下命令，赋予权限：

chmod 600 /home/git/.ssh/authorized_keys
chmod 700 /home/git/.ssh

打开我们本地cmd，输入：

ssh git@master

#master是你云服务器的ip

小插曲：我这里的master是我在我本机配置了ip映射关系,master指向的是的云服务器公网ip，由于ip地址不好记，我就做了这个映射关系，只需要再C:\Windows\System32\drivers\etc文件夹下的hosts文件内添加一条记录即可。

以上配置完成后,我们使用ssh git@master登录云服务器的git用户就不需要输入密码了。不然每次修改博客内容,再推送到云服务器上都要输入密码。

3.配置Git Hooks

sudo su #返回服务端命令行切换到 root 账户

mkdir /var/repo #mkdir /var/repo
chown -R git:git /var/repo
chmod -R 755 /var/repo #赋予权限

mkdir /var/hexo
chown -R git:git /var/hexo
chmod -R 755 /var/hexo #接下来创建 hexo 目录作为网站根目录，并赋予权限

cd /var/repo
git init --bare hexo.git 接下来创建一个空白的 git 仓库

在 /var/repo/hexo.git 下，有一个自动生成的 hooks 文件夹。我们需要在里边新建一个新的钩子文件 post-receive

vim /var/repo/hexo.git/hooks/post-receive

在该文件里面输入以下内容然后保存退出：

#!/bin/bash
git --work-tree=/var/hexo --git-dir=/var/repo/hexo.git checkout -f

chown -R git:git /var/repo/hexo.git/hooks/post-receive
chmod +x /var/repo/hexo.git/hooks/post-receive #修改权限

到这里 git仓库和钩子 已经搭建完毕了。

3.nginx安装与配置修改

我们首先需要安装nginx服务器（默认路径/etc/nginx/）

yum install -y nginx

然后我们再去编辑/etc/nginx/下的nginx.conf文件

4.推送hexo静态资源

在本地hexo根目录下的_config.yml文件内修改推送仓库地址：

deploy:
  type: git #推送方式
  #repo: git@github.com:myxym-love/myxym-love.github.io.git
  #repo: git@gitee.com:myxym-love/myxym-love.git
  repo: git@47.100.227.19:/var/repo/hexo.git #云服务器上的git仓库地址
  branch: master #推送到哪一个分支

推送：

hexo clean && hexo g && hexo d
#hexo clean清除静态资源文件,每次都执行一下，以免有缓存
#hexo g生成静态资源文件
#hexo d推送到远程仓库

5.启动nginx

systemctl start nginx #启动nginx
systemctl enable nginx #开启启动ngnix

启动成功后，访问你的域名或者ip地址就能看到内容啦。
